## v1.1.0 — 06.07.2026

### Новое

- **Selfmask — маскировка прокси под реальный сайт на вашем домене**
  - Прокси маскируется под полноценный HTTPS-сайт на собственном домене пользователя
  - Схема работы: `telemt :443 (MTProto)` → `unknown SNI / браузер` → `mask → PQ nginx 127.0.0.1:8444(порт опционально)`
  - Автоматическая установка **PQ nginx** — изолированный nginx + OpenSSL 3.5.7 со статической линковкой
    - Не заменяет системный nginx и OpenSSL
    - Устанавливается в `/opt/mtproxyl-nginx/`, работает как отдельная служба
    - Собран через GitHub Actions для `linux/amd64` и `linux/arm64`
    - Совместимость: Debian 11+, Ubuntu 20.04+
  - Поддержка постквантового обмена ключами **X25519MLKEM768** (TLS 1.3)
  - Автоматическое получение и автопродление **Let's Encrypt** сертификата через webroot
  - Конфиг nginx с 4 server-блоками: default-drop, ACME+redirect, SSL default-drop, SSL-сайт
  - 3 встроенных шаблона сайтов-заглушек:
    - **Простая заглушка** — «Сайт временно недоступен»
    - **Файловый менеджер** — форма входа с логином/паролем (всегда «неверные данные»)
    - **Cat Runner** — мини-игра (кот прыгает через кактусы)
  - Возможность указать свой URL шаблона (прямая ссылка на `index.html`)
  - Шаблоны хранятся в репозитории и скачиваются при установке, с fallback на встроенную заглушку
  - Проверка DNS (A-запись домена) перед настройкой с возможностью продолжить
  - Автоматическое применение настроек MTProxyL: `PROXY_DOMAIN`, `MASKING_HOST=127.0.0.1`, `MASKING_PORT=8444`, `UNKNOWN_SNI_ACTION=mask`
  - Защита: при активном selfmask смена домена и mask backend через обычные настройки заблокирована — только через меню selfmask
  - Проверка selfmask (verify): nginx, certbot, сертификат, backend, PQ handshake, настройки MTProxyL
  - Отключение selfmask: возвращает маскировку в обычный режим (не в `secure=true`)
  - Полное удаление PQ nginx: отдельный пункт в меню и при деинсталляции MTProxyL
  - Безопасное освобождение портов: при обнаружении системного nginx на порту 80 — запрос подтверждения, без `disable`
  - CLI: `mtproxyl selfmask status|setup|verify|disable|menu`
  - Отдельное TUI меню с полной информацией и предупреждениями PQ

- **Дополнения (утилиты) — проверка доменов на PQ-совместимость**
  - Новый раздел `[d] Дополнения` в главном меню
  - Проверка текущего SNI-домена или произвольного домена на поддержку X25519MLKEM768
  - Используется PQ OpenSSL из нашей сборки (`/opt/mtproxyl-nginx/bin/openssl`)
  - Показывает: PQ-подключение, обычное TLS, Peer Temp Key, вердикт с маркером 🟢/🟡/🔴
  - CLI: `mtproxyl pq-check [домен[:порт]]`
  - Если PQ nginx не установлен — предлагает установить

- **GitHub Actions — сборка PQ nginx**
  - Новый workflow `build-pq-nginx.yml` для автоматической сборки nginx 1.28.3 + OpenSSL 3.5.7
  - OpenSSL слинкован **статически** — бинарник не зависит от системного OpenSSL
  - Сборка на Debian 11 для максимальной совместимости с glibc
  - Поддержка `linux/amd64` и `linux/arm64` (arm64 через QEMU)
  - Публикация в GitHub Releases

### Улучшения

- **install.sh**: при деинсталляции MTProxyL теперь предлагается удалить selfmask и PQ nginx (с подтверждением)
- **install.sh**: список удаляемых компонентов обновлён — включает selfmask и PQ nginx

## v1.0.9 — 01.07.2026

### Новое

- **NFT Smart By-MEKO — Other Action** — выбор действия для non-iOS устройств (Android / Desktop)
  - `icmp-host-unreachable` — новый режим по умолчанию: сервер притворяется недоступным узлом сети, Telegram мгновенно переключается на основное соединение, медиа начинает отправляться без задержек
  - `reject (tcp reset)` — классический сброс соединения (оригинал By-MEKO), сохранён для совместимости
  - `drop` — тихое уничтожение пакета (не рекомендуется)
  - Выбор действия предлагается при первой установке Smart режима
  - Изменить действие можно в любое время: меню `[7] → [4] → [5] Other Action`
  - Доп. правила (Extra Rules) в Smart режиме автоматически наследуют выбранный тип действия
  - Текущий тип действия отображается в статусной строке NFT главного меню

- **Оптимизация системы By-MEKO** — новый пункт `[m]` в меню NFT, адаптирован из проекта MTPROTO-FIX-By-MEKO
  - TCP keepalive: `time=45 / intvl=15 / probes=3` — обнаружение мёртвых сокетов за ~90 сек вместо ~2 часов
  - Расширенные сетевые очереди: `somaxconn`, `tcp_max_syn_backlog`, `netdev_max_backlog` = 65535
  - `tcp_fastopen = 3`, `fs.file-max = 2097152`
  - Алгоритм управления перегрузкой: BBR + планировщик очереди fq
  - Полный откат к значениям до применения — все оригинальные значения ядра сохраняются
  - Автоматический откат при удалении MTProxyL (`mtproxyl uninstall`)
  - Предлагается при первой установке

- **Каталог эксперта — секция `server.listeners`** — добавлены все параметры `[[server.listeners]]` telemt
  - `ip`, `port`, `client_mss`, `announce`, `announce_ip`, `proxy_protocol`, `reuse_allow`
  - `synlimit` (false / iptables / nftables) — per-listener встроенный SYN limiter telemt
  - `synlimit_seconds`, `synlimit_hitcount`, `synlimit_burst` — generic bucket параметры
  - `synlimit_ios_seconds`, `synlimit_ios_hitcount`, `synlimit_ios_burst` — iOS-classifier bucket
  - `synlimit_hashlimit_expire_ms`, `synlimit_hashlimit_size` — параметры iptables hashlimit
  - Поддержка `[[server.listeners]]` (массив таблиц TOML) в `_apply_expert_overrides`

- **Персистентная база трафика** — трафик больше не сбрасывается при перезагрузке прокси
  - Данные накапливаются в `relay_stats/traffic_db` — дельта между сессиями суммируется
  - Отображается раздельно: «Всего с учётом перезагрузок» и «Текущая сессия»
  - Работает для общего трафика и по каждому пользователю отдельно
  - База трафика включена в бэкапы (уже покрыта директорией `relay_stats/`)

## v1.0.8 — 25.06.2026

### Новое

- **NFT Smart By-MEKO** — новый режим SYN limiter, адаптированный из проекта MTPROTO-FIX-By-MEKO
  - iOS и Android/Desktop разделяются автоматически по TTL (< 65 = iOS) на одном порту
  - iOS получает мягкий лимит (15/sec burst 30), остальные — строгий (54/min burst 1)
  - REJECT с tcp-reset вместо DROP — клиент получает мгновенный RST и переподключается за 3-8 сек вместо 10-20
  - Один порт для всех клиентов — iOS Fix v2 (отдельный порт + MSS) больше не нужен
  - client_mss в конфиге telemt не нужен
  - Доступен через меню `[7] → [s]`, CLI `mtproxyl nft smart`, пресет `mtproxyl nft preset smart`
  - При первой установке Smart предлагается по умолчанию
  - Все параметры Smart режима настраиваются (iOS rate/burst, Other rate/burst)
  - При включении Smart автоматически отключается iOS Fix v2 если был активен
- Classic режим (hard/medium/soft) сохранён для совместимости
- В меню NFT добавлено переключение между Smart и Classic режимами
- Статусная строка NFT в главном меню отображает текущий режим (Smart/Classic) и параметры
