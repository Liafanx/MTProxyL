## v1.0.8 — 25.06.2026

### Новое

- **NFT Smart By-MEKO** — новый режим SYN limiter, адаптированный из проекта MTPROTO-FIX-By-MEKO
  - iOS и Android/Desktop разделяются автоматически по TTL (< 65 = iOS) на одном порту
  - iOS получает мягкий лимит (15/sec burst 30), остальные — строгий (54/min burst 1)
  - REJECT с tcp-reset вместо DROP — клиент получает мгновенный RST и переподключается за 3-8 сек вместо 10-20
  - Один порт для всех клиентов — iOS Fix v2 (отдельный порт + MSS) больше не нужен
  - client_mss в конфиге telemt не нужен
  - Доступен через меню `[7] → [s]`, CLI `mtproxyl nft smart`, пресет `mtproxyl nft preset smart`
  - При первой установке Smart предлагается по умолчанию
  - Все параметры Smart режима настраиваются (iOS rate/burst, Other rate/burst)
  - При включении Smart автоматически отключается iOS Fix v2 если был активен
- Classic режим (hard/medium/soft) сохранён для совместимости
- В меню NFT добавлено переключение между Smart и Classic режимами
- Статусная строка NFT в главном меню отображает текущий режим (Smart/Classic) и параметры

## v1.0.7 — 25.06.2026

### Исправления

- **docker.sh**: убран дублирующий вызов `generate_telemt_config` в `run_proxy_container` — конфиг генерировался дважды подряд без причины
- **docker.sh**: добавлен `--platform` при source build движка — исправлена сборка из исходников на ARM64 VPS (aarch64)
- **settings.sh**: исправлена валидация `UNKNOWN_SNI_ACTION` — значения `accept` и `reject_handshake` больше не сбрасываются в `mask` при загрузке настроек
- **backup.sh**: исправлена `backup_create_encrypted` — путь к бэкапу теперь берётся напрямую из вывода `create_backup`, а не через `ls -1t | head -1` (устранена гонка при параллельном создании бэкапов)
- **nft.sh**: исправлено зависание при удалении MTProxyL — `ios_fix_remove` теперь принимает параметр `force` и пропускает интерактивные промпты при вызове из `nft_full_cleanup` и `uninstall`
- **tui_secrets.sh**: исправлена логика фильтрации при импорте секретов — строки с комментариями (`#`) и пустые строки теперь корректно пропускаются
- **secrets.sh**: убран скрытый побочный эффект — `load_upstreams` больше не вызывается внутри `load_secrets`; явные вызовы `load_upstreams` добавлены в `cli_main` там, где upstreams действительно нужны
- **config.sh**: устранены пустые строки в генерируемом `config.toml` — условные параметры (`ad_tag`, `proxy_secret_url`, `mask_host` и др.) теперь добавляются через `sed` только если заданы, вместо пустых `$()` подстановок в heredoc

### Новое

- **utils.sh**: реализованы CLI-обработчики для команд из README, которые ранее отсутствовали:
  - `mtproxyl port [номер]` — показать или изменить порт прокси
  - `mtproxyl ip [адрес|auto]` — показать или изменить IP для ссылок
  - `mtproxyl domain [домен]` — показать или изменить FakeTLS домен
  - `mtproxyl mask-backend [host:port]` — показать или изменить mask backend
  - `mtproxyl sni-policy [mask|drop|accept|reject_handshake]` — показать или изменить SNI-политику
- **utils.sh**: добавлена функция `validate_ip_literal` для валидации IPv4-адресов (использовалась в TUI, но не была определена)

### Улучшения

- **traffic.sh**: оптимизированы AWK-паттерны для парсинга Prometheus-метрик — паттерн `[{ ]` вместо жёсткого пробела, что предотвращает поломку при добавлении новых labels в будущих версиях telemt
- **traffic.sh**: метрики движка (`show_metrics`) теперь отображаются в рамках с box-drawing символами — улучшена читаемость разделов: соединения, upstream, пользователи, ME health, безопасность

## v1.0.6 — 24.06.2026

### Новое
- Добавлена поддержка новых параметров Telemt **3.4.19**
  - `server.client_mss_bulk` — позволяет оставить `client_mss` только на handshake, а bulk-трафик перевести на нормальный MSS
  - новая секция `[logging]` доступна в режиме эксперта
- В **режим эксперта** добавлена поддержка:
  - `censorship.tls_domains`
  - сериализации списков строк (`string[]`) в TOML
  - валидации списка доменов для `tls_domains`
- Теперь можно добавлять старые домены для ee-ссылок без смены текущего основного домена:
  - `tls_domain = "new.example.com"`
  - `tls_domains = ["old.example.com"]`

### Улучшения
- Улучшена логика работы с Docker-образами движка:
  - при выборе конкретной версии Telemt больше не используется fallback на `latest`
  - если exact-образ не найден — запускается **реальная сборка из исходников**
  - исправлена повторная сборка локального тега после неудачного/старого образа
- Улучшена работа с установкой и обновлением exact-версий telemt через GHCR

### Исправления
- Исправлена логика source build для движка Telemt
- Исправлена работа `engine update <version>` при отсутствии exact image в реестре
- Устранены ошибки с неправильным использованием старого локального Docker-тега

## v1.0.5 — 24.06.2026

### Новые возможности
- В мастер первой установки добавлено предложение включить **NFT SYN Limiter**.
  - По умолчанию предлагается включить жёсткий пресет (1/s burst 1).
  - При установке NFT Limiter теперь автоматически привязывается к публичному IPv4-адресу сервера, чтобы повысить точность и безопасность правил.
- В настройки NFT Limiter (меню `[7] → [4]`) добавлена опция **auto** для автоматического определения публичного IPv4-адреса.
- В меню бэкапов (`[9] → [3]`) добавлен интерактивный список — теперь можно восстановить бэкап, выбрав его по номеру, а не вводя полный путь к файлу.

### Улучшения
- В меню "Логи и трафик" добавлены подробные метрики движка, включая:
  - статистику соединений (autorized / rejected);
  - попытки подключения к Upstream (успешные / ошибки / rate);
  - распределение задержек (duration buckets);
  - состояние ME Health (reconnects / writers / pool drain);
  - события безопасности (desync / padding invalid).
- В меню "Движок Telemt" пункт "Список версий" теперь выводит не только теги, но и названия релизов с датами, отмечая текущую установленную версию.
- Убрано дублирование `local parsed` и блока "Безопасность" в модуле метрик.

## v1.0.0 - v1.0.4 — 23.06.2026

### Исправления
- Исправлено: `tg_connect` перенесён из секции `[timeouts]` в `[general]` — telemt выдавал предупреждение `Unknown config key`
- Исправлено: tune whitelist для `tg_connect` теперь указывает на секцию `general`, а не `timeouts`
- Исправлено: при восстановлении бэкапа теперь загружаются настройки в память (`load_settings`, `load_secrets`, `load_nft_settings`) и предлагается перезапуск прокси
- Исправлено: при восстановлении бэкапа показываются восстановленные параметры (порт, домен, количество секретов)

### Новое
- Автовыбор свободного порта метрик при установке (диапазон 9090..9199) — не конфликтует с другими приложениями на сервере
- При запуске контейнера автоматическая проверка занятости metrics port с переключением на свободный
- Порт метрик отображается в меню настроек и доступен для изменения через пункт `[g]`
- NFT limiter автоматически устанавливает `nftables` если не установлен
- Режим эксперта полностью переработан:
  - Каталог всех параметров telemt config.toml с описаниями, default, hot-reload, диапазонами
  - Интерактивный выбор раздела → параметра → карточка с описанием → ввод с валидацией
  - Override сохраняются в отдельный файл `expert.conf` с максимальным приоритетом
  - Валидация override-файла перед применением
  - Возможность открыть override-файл в nano
  - Поддержка секций: general, general.modes, general.links, general.telemetry, network, server, server.conntrack_control, server.api, timeouts, censorship, censorship.tls_fetch, access
- Новые библиотеки: `expert_catalog.sh`, `expert_mode.sh`

### Улучшения
- Функция `uninstall()` больше не чистит глобальный Docker build cache (`docker builder prune` удалён)
- Вывод `docker stop/rm/rmi` подавлен — удаление чистое без мусорного вывода
- Удаляются образы и из локального registry, и из GHCR
- Значения по умолчанию изменены: `tg_connect=30`, `client_handshake=90`, `client_keepalive=120`
- README полностью обновлён: навигация с якорями, подробное описание CLI, разделы по NFT/iOS/Expert
